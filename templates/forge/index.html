features

Migrations without the fear

Everything you need to test schema changes against real data, without touching real data.

Copy-on-write branches

A 2 TB database branches in seconds, not hours. Storage costs pennies because only changed pages duplicate.

Automatic PII masking

Emails, names, and card numbers detected and masked on branch. forge mask --audit shows exactly what changed.

Lock-time warnings

Forge estimates how long each migration locks production tables, before you merge, not after pagers fire.

CI-native

One GitHub Action. Every PR comments with its branch URL, schema diff, and migration risk score.

Query replay

Replay yesterday's production traffic against your branch. Catch the N+1 before your users do.

Zero prod access

Branches run in isolated compute. Nobody needs production credentials to test against production-shaped data.

faq

Common questions

Which databases are supported?
PostgreSQL 13+, MySQL 8+, and MariaDB today. SQL Server is in private beta, join the waitlist from your dashboard.
Does production performance suffer?
No. Branches are created from continuous WAL-based replicas, never from your primary. Production sees zero additional load.
How does masking know what's PII?
A classifier flags likely PII columns and proposes masking rules. You approve them once; they're versioned in your repo like any config.
Can I self-host?
Yes, the Forge data plane can run entirely in your VPC on Team and Enterprise plans. The control plane never sees row data.
